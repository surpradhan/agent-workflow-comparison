## Workflow Profiles

### `chain`

Four-step pipeline (decompose → retrieve → analyse → synthesise). Consistent and auditable; minimal latency overhead.

### `evaluator_optimizer`

Generator → Evaluator → Optimizer self-refinement loop. Highest quality ceiling; trades latency for accuracy.

### `human_in_loop`

Agentic workflow with human-approval checkpoints. Maximises correctness on high-stakes decisions; not fully automated.

**Notable strengths in this run:**

- Highest answer quality score: 0.836

### `manager_agent`

Manager supervises specialist sub-agents. Good at decomposable decision tasks; coordination overhead visible.

### `multi_agent_handoff`

Retrieval → Analysis → Synthesis hand-off chain. Natural fit for L3/L4 tasks; latency is additive per hop.

**Notable strengths in this run:**

- Highest tool accuracy: 77.4%

### `orchestrator_workers`

Central orchestrator fans out to worker agents. Best coverage on complex tasks; highest token cost.

### `parallel`

Concurrent sub-queries merged by a synthesis step. Excels at multi-angle analytical tasks; token cost multiplies.

### `routing`

Classifier dispatches to specialist handlers. Efficient for clearly typed tasks; overhead on boundary cases.

### `single_step`

Baseline — one LLM call, no tools. Fastest and cheapest; limited to tasks that fit in context.

**Notable strengths in this run:**

- Best at L1 (Retrieval) tasks: 100% success rate
- Best at L2 (Analytical) tasks: 100% success rate
- Best at L3 (Reasoning) tasks: 100% success rate
- Best at L4 (Decision) tasks: 100% success rate
- Most token-efficient: 1.7078 quality / 1k tokens

### `tool_using`

ReAct loop that selects and calls tools iteratively. Strong tool accuracy; latency grows with tool rounds.

**Notable strengths in this run:**
